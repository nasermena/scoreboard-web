# نقاط سكرو (Scoreboard Web)

تطبيق ويب بسيط لتسجيل نقاط لعبة **سكرو (Screw)** وإدارة الجولات واللاعبين، مع حفظ الحالة تلقائيًا عبر **localStorage**.

> الواجهة عربية (RTL) ومبنية بملفّين فقط: `index.html` و `index.js`.

## المميزات

- إعداد سريع: اختيار **عدد اللاعبين (2–6)** و **عدد الجولات (1–5)**.
- إدخال أسماء اللاعبين مع منع تكرار الأسماء.
- لوحة نقاط تعرض:
  - نقاط كل جولة لكل لاعب.
  - الترتيب الحالي (الأقل نقاطًا أولًا).
- إدخال نقاط الجولة مع تحقق صارم:
  - لا يسمح بترك أي حقل فارغ.
  - النقاط أعداد صحيحة **≥ 0**.
- **جولة دبل** (مضاعفة نقاط الجولة لكل اللاعبين):
  - خيار (Checkbox) مع رسالة تأكيد عند التفعيل.
  - تنبيه تلقائي في **الجولة رقم 4** إذا لم يتم اختيار الدبل.
- تعديل نقاط الجولة السابقة من صفحة اللوحة.
- عند انتهاء القيم:
  - عرض جدول «نقاط كل جولة لكل لاعب» + «الترتيب النهائي».
  - إمكانية تعديل **آخر جولة**.
  - بدء قيم جديد عبر تدفق واضح:
    - سؤال: هل تريد تغيير عدد اللاعبين؟ (إضافة/إزالة ضمن الحدود 2–6).
    - ثم سؤال: هل تريد تغيير عدد الجولات؟ (زيادة/تخفيض ضمن الحدود 1–5).
    - ثم زر **بدء القيم** لبدء قيم جديد بنفس الأسماء (بعد التعديل إن وُجد).
  - خيار «لا، اعرض الملخص» للانتقال مباشرة إلى صفحة الملخص.
- صفحة **ملخص** للجلسة:
  - ترتيب حسب عدد القيمات الفائزة.
  - ترتيب حسب مجموع النقاط الكلي عبر القيمات.
  - زر **بدء جلسة جديدة** (مسح بيانات الجلسة).

## التشغيل محليًا

لا يوجد بناء (Build) أو أدوات إضافية.

1. انسخ المستودع:
   ```bash
   git clone https://github.com/nasermena/scoreboard-web.git
   cd scoreboard-web
   ```
2. افتح `index.html` مباشرة في المتصفح.

### ملاحظة حول CORS

بعض المتصفحات قد تقيد العمل عند فتح الملف مباشرة (`file://`). إن واجهت مشكلة، شغّل خادمًا محليًا بسيطًا:

- Python:
  ```bash
  python -m http.server 8000
  ```
  ثم افتح:
  `http://localhost:8000`

## النشر على GitHub Pages (اختياري)

لأن المشروع Static، يمكن نشره بسهولة على GitHub Pages:

1. من إعدادات المستودع: **Settings → Pages**.
2. اختر المصدر: **Deploy from a branch**.
3. اختر الفرع `main` والمجلد `/ (root)`.

## التقنية المستخدمة

- HTML + JavaScript (Vanilla).
- TailwindCSS عبر CDN.
- أيقونة المتصفح (favicon): `Screw.ico`.

## هيكل الملفات

- `index.html` — واجهة التطبيق (RTL) + Tailwind عبر CDN.
- `index.js` — منطق اللعبة، التحقق من الإدخالات، العرض، وتخزين الحالة.
- `.gitignore` — استثناء بعض الملفات المحلية.

## التخزين المحلي (Local Storage)

التطبيق يحفظ الحالة في المتصفح باستخدام `localStorage`:

- المفتاح: `scoreboard_state_v1`
- لتفعيل/إيقاف التخزين: غيّر الثابت في أعلى `index.js`:
  - `const USE_STORAGE = true;`

زر **بدء جلسة جديدة** يقوم بمسح الحالة وإعادة التطبيق لصفحة الإعداد.

## ملاحظات حول ترتيب النقاط

الترتيب في هذا التطبيق يعتبر أن **الأقل نقاطًا هو الأفضل** (مرتب تصاعديًا). إذا أردت عكس المنطق (الأعلى أفضل)، يمكنك تعديل دالة `sortPointsAsc` في `index.js`.
